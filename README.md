# I'm Mark

Backend engineer from Melbourne with 20+ years of shipping production systems. I've been building in Spring Boot and Laravel, and lately I've been putting AI agents to work on things of questionable importance.

I like problems without a straightforward solution — Bayesian tuning, integer linear programming, multi-agent pipelines. If there's a clever algorithm hiding in a business problem, I'll find it.

I also enjoy making generative art — some examples are further down the page.

## What I've been building

| Project | |
|---|---|
| [**BitBrush**](https://github.com/mark-dingwall/BitBrush) | Real-time collaborative pixel canvas. REST APIs for canvas state, WebSocket/STOMP for live broadcasts, thread-safe placement banking. <br><sub>Spring Boot · WebSocket/STOMP · Postgres · Flyway · Fly.io</sub> |
| [**Mystery-Manager**](https://github.com/mark-dingwall/Mystery-Manager) | Allocates bulk produce overage into mystery boxes that customers actually want. Bayesian hyperparameter tuning picks the weights, an ILP solver does the allocation, and a glass-box ML model finds the gaps. <br><sub>Python · Optuna · PuLP/HiGHS · LightGBM · Bradley-Terry</sub> |
| [**apples-to-apples**](https://github.com/mark-dingwall/apples-to-apples) | AI-driven pricing pipeline. Playwright feeds 3 LLM stages with 8 parallel agents. Deterministic cross-validation of output with graceful fallback, trend analysis, and audit trail. <br><sub>Python · Playwright · LLM orchestration</sub> |
| [**Zeroshot**](https://github.com/mark-dingwall/zeroshot) *(fork)* | Extended an open-source multi-agent engine with parallel analyst clusters, real-time subagent tracking, and parameterised templates. Built a framework-agnostic quality gate system. <br><sub>JavaScript · Multi-agent architecture</sub> |
| [**Guestflow**](https://github.com/mark-dingwall/Guestflow) | Sync middleware for Guesty and Webflow. Live pricing, calendar, and booking data across distributed systems. Two-tier cache with refresh locking handles a 5-token/day rate limit without thundering-herd blowups. <br><sub>TypeScript · Distributed caching · REST APIs</sub> |
| [**mark-dingwall.github.io**](https://github.com/mark-dingwall/mark-dingwall.github.io) | Personal site with portfolio, creative coding sketches (Processing), and a multiplayer online canvas. <br><sub>JavaScript · Processing · GitHub Pages</sub> |
